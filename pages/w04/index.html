Forms

There are a lot of types of inputs. These are a few:

  • <input type="text">
  • <input type="button">
  • <input type="checkbox">
  • <input type="date">
  • <input type="email">
  • <input type="hidden">
  • <input type="image">
  • <input type="number">
  • <input type="password">
  • <input type="radio">
  • <input type="submit">
  • <input type="tel">
    • There is even an input for color! 
  • <input type="color">

There are also <textarea> for larger inputs, <select> for dropdown, etc. All these elements, as with every HTML element, have very useful attributes.

  • Value: It's the value entered by the user
  • Placeholder: A prompt for the user to complete the field
  • Min/max: To limit the range of data in inputs of type number
  • Required: For mandatory inputs

Object Oriented Programming

There are three main concepts in OOP: encapsulation, polymorphism and inheritance.

Encapsulation

Encapsulation is keeping hidden inside the objects all the complex logic and providing easy ways to work with these objects.

Polymorphism

Several objects can share methods with other objects besides having different properties. Nevertheless, each object has the ability to override any method that needs a more specific implementation.

Inheritance

One object can inherit the attributes and methods of another object. Thus we can create an improved version of the functionalities of the older object when writing the newer one.

Classes

Another vital part of OOP are classes. Classes are model objects, or blueprints for objects. Then, objects are instances of that class, inheriting its attributes and methods.
