Client-side form validation

Client-side form validation is when the client/browser validates a form before sending it to the backend for validation. Validating the user's input before sending it to the backend, we allow the user to fix it right away instead of waiting and then getting an error.

Client-side validation, though, should not be the only validation since it is easy to bypass. The most exhaustive form validation for user attacks should be on the server side.

These are a few examples of client-side form validation extracted from MDN:

  • "This field is required" (You can't leave this field blank).
  • "Please enter your phone number in the format xxx-xxxx" (A specific data format is required for it to be considered valid).
  • "Please enter a valid email address" (the data you entered is not in the right format).
  • "Your password needs to be between 8 and 30 characters long and contain one uppercase letter, one symbol, and a number." (A very specific data format is required for your data).

Fetch API

The Fetch API provides a JavaScript interface for accessing and manipulating requests and responses. It also provides a global fetch() method that provides an easy way to get resources asynchronously across the network.

Example of basic request:

fetch('http://example.com/movies.json')
  .then((response) => response.json())
  .then((data) => console.log(data));
